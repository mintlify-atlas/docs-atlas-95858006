redb is an embedded key-value database written in pure Rust. It stores data in a collection of copy-on-write B-trees and provides fully ACID-compliant transactions, zero-copy reads, and MVCC-based concurrency — all with no external dependencies.

## Why redb?

redb is designed for applications that need an embedded database without the complexity of a client-server setup. It is loosely inspired by [lmdb](http://www.lmdb.tech/doc/) but written entirely in safe Rust with a modern, ergonomic API.

<CardGroup cols={2}>
  <Card title="ACID transactions" icon="shield-check">
    Every write is atomic, consistent, isolated, and durable. Transactions either commit fully or roll back completely.
  </Card>
  <Card title="MVCC concurrency" icon="arrows-split-up-and-left">
    Multiple readers run concurrently alongside a single writer, with no blocking between them.
  </Card>
  <Card title="Zero-copy reads" icon="bolt">
    Data is accessed directly from the memory-mapped file — no deserialization overhead for reads.
  </Card>
  <Card title="Crash safety" icon="circle-check">
    All pages are checksummed. Partially-written transactions are detected and rolled back on next open.
  </Card>
  <Card title="Savepoints" icon="bookmark">
    Create named snapshots of database state and roll back to them at any time, even across restarts.
  </Card>
  <Card title="Custom backends" icon="plug">
    Swap out the storage layer with your own implementation of the `StorageBackend` trait.
  </Card>
</CardGroup>

## Quick example

```rust quickstart.rs
use redb::{Database, Error, ReadableDatabase, TableDefinition};

const TABLE: TableDefinition<&str, u64> = TableDefinition::new("my_data");

fn main() -> Result<(), Error> {
    let db = Database::create("my_db.redb")?;

    let write_txn = db.begin_write()?;
    {
        let mut table = write_txn.open_table(TABLE)?;
        table.insert("my_key", &123)?;
    }
    write_txn.commit()?;

    let read_txn = db.begin_read()?;
    let table = read_txn.open_table(TABLE)?;
    assert_eq!(table.get("my_key")?.unwrap().value(), 123);

    Ok(())
}
```

## Get started

<Steps>
  <Step title="Add redb to your project">
    Add the dependency to `Cargo.toml`:
    ```toml
    [dependencies]
    redb = "4"
    ```
  </Step>
  <Step title="Define your tables">
    Declare typed table definitions at compile time:
    ```rust
    use redb::TableDefinition;
    const USERS: TableDefinition<u64, &str> = TableDefinition::new("users");
    ```
  </Step>
  <Step title="Open a database and write data">
    Create a database file, start a write transaction, and insert records:
    ```rust
    let db = Database::create("app.redb")?;
    let txn = db.begin_write()?;
    {
        let mut table = txn.open_table(USERS)?;
        table.insert(&1, "Alice")?;
    }
    txn.commit()?;
    ```
  </Step>
  <Step title="Read data back">
    Open a read transaction and query your table:
    ```rust
    let txn = db.begin_read()?;
    let table = txn.open_table(USERS)?;
    println!("{}", table.get(&1)?.unwrap().value());
    ```
  </Step>
</Steps>
